# Norm loss (per-weight-group L2-norm 正則化) の導入

- Status: Accepted
- Date: 2026-06-02

## Context

nnue-pytorch との Elo gap を詰める init / loss alignment の一環として、Ranger21 が
備える norm loss (Georgiou et al. 2021, "Norm Loss: An efficient yet effective
regularization method for deep neural networks") を導入する。norm loss は各 weight
group の L2 norm を 1 (oblique manifold) へ緩く引き寄せる weight decay 代替で、
過学習抑制と汎化改善を狙う。

reference の `nnue-pytorch` / nodchip fork が vendor する `ranger21.Ranger21` +
`model/lightning_module.py` は、stale-`p` バグにより norm loss が FT (`input`) weight
(= FT 本体 + PSQT 列) には掛からず隠れ層 (L1f/L1/L2/L3) + 全 bias にのみ実効的に掛かる
(後述「reference 調査」)。tatara はこの実効挙動をなぞらず、Georgiou / Ranger21 が
intended した形 — FT/PSQT を含む全 weight group への一様適用 — を採用し、棋力を SPRT で
単独評価する。

## Decision

`--norm-loss` (default OFF) で opt-in する per-weight-group の L2-norm 正則化を
LayerStack trainer の optimizer step に追加する。強度は `--norm-loss-factor`
(既定 1e-4) で設定する。default OFF では norm loss kernel を一切 launch せず
baseline と bit-identical。

### 式

各 weight group `g` (= 1 出力ニューロンに流れ込む重みベクトル、または bias
テンソル全体) について step ごとに 1 回:

```
norm_g = ||w_g||_2
corr_g = 2 * factor * (1 - 1 / (norm_g + eps))
w_g   *= 1 - lr * corr_g
```

`norm_g > 1` は縮小、`< 1` は拡大で、両側から `norm_g = 1` (oblique manifold)
へ緩く引き寄せる。`lr` は schedule 後の実 lr、`eps` は optimizer eps を流用
(div-by-zero ガードのみで、norm が O(1) のため値の差は無視できる)。

### reference 調査 (norm loss が掛かるのは FT weight 以外)

reference 実装 (`nnue-pytorch` master / nodchip fork、`ranger21.Ranger21` +
`model/lightning_module.py` + `model/modules/feature_transformer/module.py`) を
精査した結果、norm loss は **FT (`input`) weight 1 つだけが対象外**で、それ以外には
実効的に掛かっていると判明した。本実装はその実効挙動 (隠れ層 + bias) に沿いつつ、
漏れている FT/PSQT も含めて intended に一様適用する位置づけ (棋力は SPRT で単独評価)。

調査結果:

1. **vendored Ranger21 の stale-`p` バグ**: nnue-pytorch (official-stockfish /
   nodchip 両 fork とも同一) の `ranger21.py` では normloss ブロックが step() 内
   phase-2 の `for group in self.param_groups:` には居るが **inner の
   `for p in group["params"]:` ループの外**にあり、その時点で束縛されている
   stale な `p` (= 前 group / phase-1 で最後に処理した param) に適用される。9 個の
   param group を辿ると、normloss は **`input.weight` を除く全 param** (L1f/L1/L2/L3
   の weight・bias、FT bias) に正しい `unit_norm` で 1 step 1 回掛かり、**`input.weight`
   だけが一度も対象にならない** (group 0 の先頭 param で stale `p` になり得ないため)。
   nnue-pytorch では `input.weight` が FT 本体 + PSQT 列 (`L1..L1+num_psqt`) を物理的に
   含むので、結果として **FT 本体と PSQT の両方が norm loss を受けない唯一のテンソル**
   になる。
2. **upstream は修正済**: 現行 lessw2020/Ranger21 (main) は normloss を per-param
   helper `apply_weightdecay_normloss_updates(p, ...)` に切り出し inner ループ内で
   呼ぶため全 param に正しく掛かる。nnue-pytorch が古い vendored copy を抱えているの
   が原因。official-stockfish master (2026-01-04) / nodchip とも config 上は
   `normloss_active=True` (factor 1e-4) で「効かせているつもり」。
3. **仮に FT も正しく適用しても軸が異なる**: nnue-pytorch の FT (`input`) weight は
   `BaseFeatureTransformer` で `torch.rand(num_inputs, num_outputs)` =
   `[ft_in, ft_out]` の **転置** layout (隠れ層 `nn.Linear` の `[out, in]` とは逆)。
   `unit_norm` は 2D に `dim=1` を使うので、FT では **per-input-feature** (ft_out
   方向の norm) になる。「2D=per-output-neuron」が成立するのは `[out, in]` の隠れ層
   だけ。ただし上記 1 で FT は実際には対象外なので、これは仮定上の注記。

### 対象層と norm 粒度 (intended Georgiou/Ranger21 仕様)

tatara は intended な oblique-manifold (各出力ニューロンの fan-in ベクトルを単位
norm へ) を **全 weight tensor に一様に per-output-neuron** で適用し、bias は
per-tensor scalar とする:

| tensor | レイアウト | norm 粒度 |
|---|---|---|
| FT weight `[ft_in, ft_out]` | strided column | per-output-neuron (ft_out 個) |
| L1f weight `[ft_out, l1_out]` | strided column | per-output-neuron (l1_out 個) |
| L1 weight `[bucket*l1_out, ft_out]` | contiguous row | per-output-neuron |
| L2 weight `[bucket*l2_out, l2_in]` | contiguous row | per-output-neuron |
| L3 weight `[bucket*1, l2_out]` | contiguous row | per-output-neuron |
| PSQT weight `[ft_in, num_buckets]` | strided column | per-output-neuron (num_buckets 個) |
| 各 bias (FT/L1/L1f/L2/L3) | 1D | per-tensor scalar |

PSQT も同じ intended 一様適用として per-output-bucket (bucket 列ごとに ft_in 要素、
`psqt_w[feat*num_buckets + bucket]`) で対象に含める。これは reference 整合ではなく
「intended norm loss を全 weight に一様適用する」設計判断による。Simple アーキは
nnue-pytorch 対応物でなく検証経路も別のため `--norm-loss` を reject する
(LayerStack 専用)。

注: FT を per-output-neuron にするか (本実装)、上記調査 3 の per-input-feature に
するかは intended 解釈の分岐点。Georgiou 論文の oblique manifold は「各ニューロンの
重みベクトル」を単位 norm 化する趣旨で、本実装は全層 per-output-neuron で統一した。
SPRT で効果が出ない / 悪化する場合は per-input-feature FT も再評価候補。

### generic group indexing

3 レイアウトを `(n_groups, group_pitch, elem_stride, group_len)` の 4 値で統一
表現し、kernel を 1 対 (`norm_loss_reduce` + `norm_loss_apply`) に集約する:

```
off(g, pos) = g * group_pitch + pos * elem_stride   (g < n_groups, pos < group_len)
```

- contiguous row `[n_groups, group_len]`: `pitch=group_len, stride=1`
- strided column `[group_len, n_groups]`: `pitch=1, stride=n_groups`
- per-tensor scalar: `n_groups=1, pitch=0, stride=1, group_len=テンソル長`

`norm_loss_reduce` は 2D grid (x=group, y=pos チャンク) で各 group の部分 Σw² を
`norm_scratch` へ `atomicAdd` し (group_len が巨大な FT で occupancy を稼ぐため)、
`norm_loss_finalize` が per-group に sqrt して L2 norm にする。`norm_loss_apply` は
1 thread = 1 weight で補正を乗ずる。`norm_scratch` (長さ = 対象中の最大 group 数) は
同 stream 上で tensor ごとに 0fill → reduce → finalize → apply の順で再利用する
(0fill が atomicAdd の前提、次 tensor の 0fill 前に当該 apply が読み終える)。
atomicAdd の加算順序は非決定で reference CPU と bit 一致しないため、GPU↔CPU 等価性は
相対許容 (`assert_close_rel`) で検証する。

### 適用順序 (radam step の前)

norm loss は radam optimizer step の **前** に適用する:

1. radam の per-layer 量子化 clamp が最後の演算になり clamp 不変条件を保つ
   (norm loss が clamp 後に微小に範囲外へ押し出すのを避ける)。
2. FT の FP16 mirror (`ft_w_h`) は後続 radam / lookahead が master から再同期する
   ため、norm loss 側で mirror を更新する kernel variant が不要になる。
3. nnue-pytorch の "decay/normloss を step に先立ち p.data に適用" する順序とも
   整合する。

既定 no-op の `--weight-decay` とは独立に併用可能。両者とも weight への乗法縮小だが、
decay は radam step 内 (`1 - decay*rate`)、norm loss はその前段。

## Alternatives considered

- **per-tensor (Frobenius) norm に統一**: kernel はレイアウト非依存で最も単純だが、
  Georgiou / Ranger21 の linear-layer 既定 (per-output-neuron) と異なる。採用せず。
- **対象を dense weight (L1/L2/L3) のみに最小化**: kernel surface は減るが、
  intended 仕様は FT / L1f / bias も対象に含むため、"最適な状態を実験で確かめる"
  という目的に反する。採用せず (全層に一様適用)。
- **radam step の後に適用 + FT mirror 専用 variant**: clamp 不変条件を崩し、
  mirror 同期 kernel が 1 種増える。前段適用で両方回避。

## Consequences

- opt-in 時、対象 weight に対し reduce + apply の 2 pass が増える。最大の FT
  weight (`ft_in * ft_out`) で per-step に追加 read 1 + read/write 1 pass。
  棋力検証用の opt-in flag なので throughput は許容範囲とする。
- 数値検証: CPU reference (`gpu_kernels::pointwise::norm_loss`) の unit test と、
  3 レイアウトを covering する GPU↔CPU 同等性テスト
  (`norm_loss_reduce_matches_cpu` / `norm_loss_apply_matches_cpu`)。
- 棋力可否は held-out 400 sb + SPRT で長期 run により判定する (baseline =
  現行ベスト ft-fanin / l1l2l3-fanin の勝者)。
- experiment.json に `norm_loss_factor` を記録 (有効時のみ) して再現性を確保。
